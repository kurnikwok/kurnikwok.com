AI shaping glossary

Bite-sized definitions for the simple mechanism and concept stack used across the AI shaping papers and public pages.

The glossary explains how AI shaping moves from one-off prompting to reusable AI work patterns that carry context, source basis, domain-practice guidance, subject-specific context, review boundaries and work state under human direction.

AI shaping
The operating-model discipline for giving broad, general-purpose AI capability a practice-guided, subject-aware work pattern and work state so useful output can become grounded, reviewable, repeatable and resumable productive AI work under human direction. In simple terms, it moves from one-off prompting to reusable AI work patterns that carry context, standards, source basis, review rules and work state forward.
AI-shaping intelligence
The reusable meta-capability that builds, improves and renews its own shaping pattern so that shaped intelligence can carry more recurring work burden in a domain. In simple terms, it is the capability for using AI, under human direction, to improve the reusable AI work pattern that future AI-enabled work uses.
Shaped intelligence
The domain-facing reusable capability that carries more of the recurring work burden in a work domain, including domain-practice application and subject-context reasoning, while human direction, judgement and action remain explicit. In simple terms, it is AI using a reusable, domain-facing work pattern so the person does not rebuild the same context, standards, review boundaries and next-action logic every time.
Project-managing shaped intelligence
The public capability-evidence instance of shaped intelligence applied to project and business-as-usual (BAU) IT support work, carrying recurring project-support work burden while human authority remains explicit.
Domain-practice guidance
Guidance used to shape AI-enabled work for a recurring work domain, including relevant work-practice standards, review expectations and operating patterns.
Domain-practice application
The use of relevant work-practice standards, review expectations and operating patterns inside shaped intelligence, so AI-enabled work follows the domain’s work pattern rather than a generic response pattern.
Subject-specific context
The matter-specific information the AI-enabled work depends on, such as the project, issue, stakeholder, support context, constraints or supplied facts.
Subject-context reasoning
The use of subject-specific project, issue, stakeholder or support-context information to frame AI-enabled work in a reviewable way.
Work pattern
The shaped role, source basis, domain-practice guidance, subject-specific context, review boundaries and output expectations for a recurring kind of work. In simple terms, it is the reusable instruction-and-context bundle that helps AI handle the same kind of work consistently instead of starting from a fresh prompt each time.
Work state
The preserved position of the AI-enabled work: what happened, what changed, what is unresolved, what needs review and what happens next, so the work can resume without rebuilding the whole work basis.
Productive AI work
AI-enabled work that is grounded, reviewable, repeatable and resumable under human direction; the work outcome of AI shaping that can be reviewed, approved, continued and reused.
Work burden
The recurring human load required to turn useful but fragile AI output into productive AI work, including context rebuilding, source-basis repair, review-state preservation and resumption work.
Work-burden shift
The visible before/after change where more recurring work burden moves from the responsible person to shaped intelligence, including reconstruction, domain-practice application, subject-context reasoning, review/resumption and communication-preparation work, while judgement, approval and real-world action remain human-owned.
Reconstruction
The repeated work of rebuilding the AI work basis before AI-enabled work can continue: context, source material, prior decisions, constraints, risks, review status and next action.
Resumption
Continuing AI-enabled work from a preserved reviewable state after a pause, handover, context loss or new information, instead of asking AI to restart from a rebuilt prompt.
Context rebuilding
A plain-language version of reconstruction. In ordinary AI use, this often means writing a long or complex prompt to re-explain the work before the AI can help again.
Operating-model design
The broader discipline of arranging roles, controls, workflows, review boundaries and repeatable patterns so that work can be performed safely and consistently.
Grounded
AI-enabled work is grounded when it remains tied to source basis, context, review state and relevant constraints rather than becoming detached AI output.
Reviewable
AI-enabled work is reviewable when it is structured so human judgement, approval, correction, disclosure and accountability remain explicit.
Repeatable
AI-enabled work is repeatable when it can recur through a reusable work pattern without the person rebuilding the whole work basis each time.
Resumable
AI-enabled work is resumable when it can continue after pause, handover or new information while preserving work state and next-action logic.
Public-stage
The public reading or evaluation stage for AI-shaping material, limited to category understanding or bounded capability evaluation without implementation reliance.
Public-stage material
Material that supports public category understanding or bounded public-stage capability evaluation of AI shaping, without providing method transfer or deployment assurance.
Protected-stage
The non-public stage required before AI-shaping implementation-level detail, method transfer or stronger assurance questions can be reviewed.
Protected-stage discussion
A non-public review context required for AI-shaping implementation-level detail, method transfer or stronger assurance questions.
